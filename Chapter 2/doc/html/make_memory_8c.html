Detailed Description

Functions about memory allocating.

Author
lgxZJ.nosp@m.@out.nosp@m.look..nosp@m.com
Date
5/14/2016 
    This module provide a way that users can allocate
    memory in heap, search certain memory blocks and
    free all memory at a time. Indeed, once one memory
    block is allocated, it is also recorded internally.
    Thus, search like functions can be done with this
    information.
Note
This should be the base module of memory allocation, another other module shall use features provided by this module.
Warning
The test case may not be enough, if some bugs were found, please contact me!

void* findMemoryBlock ( MemoryType  type)

Find a certain memory block.

Parameters
[in]typeThe type of memory block to be found.
Returns
NULL for not found, non-NULL for found.
Remarks
Users can pass the same memory type to this function to traverse all the memory blocks with type type, and the traverse ends when it returns NULL. Once users use a new memory type, it begins a new searching.
Note
Users can search MEMORY_TYPE_NONE memory blocks, but be sure to not change the contents of those, for it is internally used and may cause problems.
See also
makeMemoryBlock, freeMemoryChain.

void* makeMemoryBlock ( unsigned  bytes,
MemoryType  type 
)

Allocate a memory block of a certain memory type which can be used later for searching.

Parameters
[in]bytesThe bytes of memory to be allocated.
[in]typeThe memory type of allocated memory which must be registered first. To register memory types, use registerUniqueType().
Returns
If the function succeeds, the return value is a pointer to the created memory block tag with type type. If the function fails, the return value is NULL which indicates there is no enough memory.
Remarks
Any memory allocated by this function is internally traced. When finish using all these memory blocks, call freeMemoryChain() to release them. One should always use one memory type to allocate memory for the same memory content.
Warning
User must not free these blocks through c function free(), the only way to free is to use freeMemoryChain().
Note
The caller can use MEMORY_TYPE_NONE as the memory type, but be careful!. For details, see MEMORY_TYPE_NONE.
See also
freeMemoryChain, findMemoryBlock, MemoryType, MEMORY_TYPE_NONE.
int registerMemoryType ( MemoryType  type)

Try register a memory type for later memory allocating.

Parameters
[in]typeA memory type to be registered. It can be MEMORY_TYPE_NONE or any integer value.
Returns
If the function succeeds and the memory type is registered successfully, the return value is MT_SUCCESS. If the function succeeds but memory type is already registered, the return value is MT_EXISTS. If the function fails, the return is MT_ERROR.
Remarks
Users can try to register with any interger value, once succeeds, it can be used for later memory allocating.
Note
It is heavily not recommended to use because of it is verbose, try to use registerUniqueType() instead.
See also
MEMORY_TYPE_NONE, registerUniqueType, MT_SUCCESS, MT_ERROR, MT_EXISTS, makeMemoryBlock.
